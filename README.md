## What you get

**Root Record Weather Manager** brings public weather and hazard data into a **local-first** app on your PC: named **saved locations**, **U.S. and Canada** weather **alerts** where applicable, **earthquake** and **tsunami** feeds, **forecasts** and **imagery** (radar / satellite) where the app is configured to show them, and extra **hazard** context (e.g. wildfire, tropical tracking) with links to authoritative sources.

Your **locations, preferences, and database** stay **on your device** (under your Windows user profile). You can use a **local session** without signing in; signing in is **optional** and enables plan features, faster live refresh where your tier allows, and **Root Record**-aligned services when you choose them. **In-app “About” and “Settings”** explain the exact options on your build.

## System requirements

| | |
| --- | --- |
| **OS** | **Windows 10** or **Windows 11**, **64-bit (x64)** |
| **Storage** | Enough space for a normal desktop install (installer size is on each [release](https://github.com/RootRecord/rootrecord-weather-manager-download/releases) page) |
| **Network** | Required for **live** maps, alerts, and feeds; your stored history is still on disk when the network is off |

**Signed** Windows installers (when we publish with Authenticode) are easier to run on strict PCs; see each release for what was shipped.

## Updates

Installed copies use **in-app** updates: the app looks at this repo’s **Releases** and prompts you to **download** and **restart** when a newer version is available. You can also **check manually** (for example from **About → Check for updates…**). Development builds and side-loaded copies may disable or skip that path.
